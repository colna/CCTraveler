## 测试场景

### 场景 1: 全链路规划思维测试

**输入**: "五一想去遵义玩 3 天"

**期望行为**:
1. ✅ 主动询问：出发地、人数、预算
2. ✅ 告知当前版本不支持交通查询
3. ✅ 询问交通方式和费用
4. ✅ 根据剩余预算推荐酒店
5. ✅ 给出区域建议
6. ✅ 提供简单行程建议

### 场景 2: 主动询问测试

**输入**: "帮我查一下上海明天的酒店"

**期望行为**:
1. ✅ 主动询问：从哪出发、预算、星级/区域要求
2. ✅ 先用 search_hotels 查询本地数据
3. ✅ 如无数据再调用 scrape_hotels
4. ✅ 用中文总结结果，推荐前 3 个

### 场景 3: 工具约束测试

**输入**: "爬取上海酒店，max_pages=10"

**期望行为**:
1. ✅ 自动限制为 5 页（不会爬取 10 页）
2. ✅ 验证日期格式
3. ✅ 检查日期合理性

### 场景 4: 超出能力范围测试

**输入**: "帮我订这个酒店"

**期望行为**:
1. ✅ 明确告知：只能提供信息和建议，不能直接预订
2. ✅ 给出替代方案：通过携程 App 预订
3. ✅ 保持产品定位：我的价值是帮您找到最优方案

### 场景 5: 参数验证测试

**输入**: "搜索酒店，min_star=6"

**期望行为**:
1. ✅ 返回错误：星级必须在 1-5 之间

**输入**: "搜索酒店，max_price=100, min_price=200"

**期望行为**:
1. ✅ 返回错误：最高价格不能低于最低价格

### 场景 6: 文件大小限制测试

**输入**: "导出所有数据为 JSON"（假设数据量超过 50MB）

**期望行为**:
1. ✅ 返回错误：导出文件过大，超过 50MB 限制
2. ✅ 提示：请使用筛选条件减少数据量

## 实施总结

### 已完成 ✅

1. **System Prompt 完全重构**
   - 角色定位从"酒店价格情报"升级为"全链路出行规划助手"
   - 添加完整的强制规则和推荐规则
   - 提供 9 个典型场景的执行流程模板

2. **工具定义增强**
   - 所有工具添加 JSON Schema 约束
   - 添加详细的限制说明
   - 参数范围明确定义

3. **执行器防护**
   - 4 个工具处理函数全部添加参数验证
   - 日期、价格、星级、评分等关键参数的合理性检查
   - 文件大小、数组长度等资源限制

4. **配置优化**
   - 禁止并发爬取（max_concurrent=1）
   - 添加详细注释说明

### 核心改进点

1. **产品定位对齐** ⭐
   - Agent 不再只是"查酒店"，而是"规划出行"
   - 主动询问交通、预算、行程
   - 体现"私人旅行顾问"的定位

2. **行为边界清晰** ⭐
   - 明确当前能力和未来能力
   - 超出范围时给出友好提示和替代方案
   - 不做承诺无法实现的功能

3. **安全防护到位** ⭐
   - 多层验证：JSON Schema + 运行时检查
   - 资源限制：爬取频率、文件大小、数组长度
   - 隐私保护：不记录敏感信息

4. **用户体验优化** ⭐
   - 主动询问而非被动响应
   - 多方案对比和推荐理由
   - 容错降级和友好错误提示
